simple

com.apama.adbc.SQLStatement static simple(string sql, sequence<any> parameters)
Execute a single SQL query or update command.
Parameters:
sql
parameters - Parameters for "?" placeholders if the sql is to be sent to the database as a prepared statement (which provides protection against SQL injection attacks and faster execution if the query will be repeated later.

If no parameters are provided, a non-prepared statement is used. Non-prepared statements are typically quicker for single use (and slower for repeated uses), but you need to be extremely careful to validate and escape any parameters in the sql string to avoid the risk of a SQL injection security vulnerability; if in doubt specify your parameters here rather than manually constructing a SQL string.

withResultsInOneEvent

com.apama.adbc.SQLStatement withResultsInOneEvent(integer maxRows)
Results all results in a single SmallResultsSet instead of one XXX event per row. It is mandatory to set maxRows when using this to limit the size of this event, since large Apama events are not recommended.
